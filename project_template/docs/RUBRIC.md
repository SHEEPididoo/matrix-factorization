## 项目评分 Rubric（建议版）

总分 100。你也可以按课程侧重点调权重。

---

## 1) 可复现性与工程规范（20）

- **环境与运行说明（8）**
  - README 清晰：数据放置、训练、评估、启动 demo 的命令齐全
- **目录与产物规范（6）**
  - `data/`、`features/`、`artifacts/` 职责清晰；大文件不进 git
- **缓存与性能意识（6）**
  - embedding/LLM 结果落盘缓存；启动/推理时间可接受

---

## 2) 数据理解与问题定义（15）

- **问题定义清晰（5）**：推荐对象/用户目标/业务约束明确
- **探索与清洗到位（10）**：稀疏度、长尾、冷启动比例、切分策略合理

---

## 3) Baselines 与模型实现（25）

- **至少 2 个经典模型（15）**
  - 示例：MF + CF / Baseline + CF / Content-based + MF
- **实现质量（10）**
  - 训练/预测流程正确、参数可配置、结果可复现

---

## 4) LLM/Embedding 增强与 Ablation（20）

- **增强设计合理（10）**
  - embedding 做召回/混合；或 LLM 抽取标签做可解释特征
- **Ablation 对比（10）**
  - 至少给出“无增强 vs 有增强”的离线指标对比，并解释原因

---

## 5) 评估与分析（10）

- **离线指标完整（6）**
  - Top-K：Precision/Recall/NDCG 至少两个；显式评分可选 RMSE
- **失败案例分析（4）**
  - 至少 2 个失败样例 + 可能原因 + 可改进点

---

## 6) Demo 与表达（10）

- **可交互 demo（6）**：自由文本输入或用户选择 → 返回推荐
- **解释与表达（4）**：解释推荐理由，清晰讲述取舍与限制
